Anatomía del mockup

Seis pantallas agrupadas por versión de release:

  • v0Login y Chat con el main — la base mínima (auth + chat).
  • v1Lista de proyectos y Sesión activa — el reemplazo del SSH para coding desde la oficina.
  • v2Mini-agentes y Topics — orquestación visible y memoria del main.

Patrones visuales clave

  • Sidebar fijo con navegación entre secciones + badge contador (proyectos, mini-agentes, topics).
  • Topbar contextual con breadcrumb y botones de acción específicos por pantalla.
  • Topic pills rosa al pie de mensajes del main — indica el contexto cargado / delegación silenciosa.
  • Tool calls con borde naranja en sesiones de coding — visibilidad de qué hace Claude por dentro.
  • Dark mode permanente, monospace (JetBrains Mono), acentos teal/cyan/pink/indigo.
  • Sidebar contextual en sesiones de proyecto — muestra hermanas + botón "volver".

Decisiones que se traducen al frontend

  • Stack: Vite + React 19 + Tailwind v4 + shadcn/ui · cookie httpOnly para JWT.
  • Conexiones: WS /ws/agent para el chat principal · WS /ws/project-session/<id> para sesiones de coding.
  • Cliente sin estado: todo viaja por WebSocket. Recargar página NO pierde nada (la session vive en server).
  • Componentes mínimos: chat panel, message list, composer, project card, agent card, topic card. Sin templates pesados.
